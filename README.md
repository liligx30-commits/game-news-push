# 资产语义搜索工具

基于 Kimi 大模型的游戏资产智能打标与语义搜索系统。

## 快速开始

### 1. 安装依赖
```
pip install -r requirements.txt
```

### 2. 配置
编辑 `config.json`：
- `kimi_api_key`：Kimi API 密钥（已配置）
- `source_dir`：要处理的资产目录（已指向测试文件）

### 3. 一键启动
双击 `start.bat`，选择操作：
- **1 打标**：用 Kimi 为所有图片生成中文描述和标签
- **2 搜索**：启动搜索服务，浏览器打开搜索界面
- **3 打标+搜索**：先打标再启动搜索

## 使用流程

### 第一步：打标（数据处理）
```
python tag_assets.py          # 增量打标（跳过已有标签的文件）
python tag_assets.py --force  # 强制重新打标全部文件
python tag_assets.py --dry-run # 预览将要处理的文件
```

Kimi 会为每张图片生成一段 250 字以内的中文描述，同时从文件名自动提取初始标签。PSD 文件会生成占位描述，等你手动编辑。

### 第二步：搜索（检索服务）
```
python search_server.py
```

浏览器自动打开搜索界面 → 输入关键词（如"猫"、"萝卜"、"国风建筑"）→ 查看结果 → 点击「打开位置」直接在资源管理器中定位文件。

### 标签编辑
- 界面内：点击卡片上的「+ 编辑」按钮修改标签和描述
- 手动编辑：直接修改 `tags.json` 文件，保存后刷新页面生效

## 搜索技巧
- 单字关键词：猫、花、树、桥
- 多词组合：萝卜 雕像、江南 建筑
- 语义描述：破旧道具、红色建筑、水上景观
- 文件名搜索：输入文件名的一部分

## 文件说明
| 文件 | 用途 |
|------|------|
| `config.json` | API 配置、源目录路径 |
| `tag_assets.py` | 打标脚本 |
| `search_server.py` | 搜索服务后端 |
| `static/index.html` | 搜索界面 |
| `tags.json` | 标签数据库（可手动编辑） |
| `start.bat` | 一键启动脚本 |

## 标签格式 (tags.json)
```json
{
  "文件的绝对路径": {
    "description": "Kimi 生成的中文描述",
    "tags": ["标签1", "标签2"],
    "last_tagged": "2026-06-30 12:00:00"
  }
}
```
